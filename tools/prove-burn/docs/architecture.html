Secret Structure

The secret is constructed from 4 components to ensure uniqueness and prevent replay attacks:

termCode
1 byte
Termination code
chainCode
1 byte
Destination chain ID
xfgTxHash
8 bytes
XFG transaction hash
randomSalt
22 bytes
Random entropy
secret = termCode || chainCode || xfgTxHash || randomSalt

Why Include XFG Transaction Hash?

Security Benefits:
  • Unique Nullifier: Each burn has a unique nullifier
  • Replay Protection: Prevents double-minting attacks
  • 1:1 Mapping: Links specific Fuego transaction to mint
  • Verifiability: Can prove burn occurred without revealing details

Circuit Design

// Public Inputs (revealed) nullifier = hash(secret) commitment = hash(nullifier) recipient_addr_hash = keccak256(recipient_address) // Private Inputs (hidden) secret = termCode || chainCode || xfgTxHash || randomSalt fuego_block_height fuego_block_hash // Constraints 1. nullifier = hash(secret) 2. commitment = hash(nullifier) 3. block_height remains constant 4. block_hash remains constant 5. recipient_hash remains constant
